# Implementation Plan

- [x] 1. タブ統合とモジュールの土台構築
- [x] 1.1 既存タブシステムにダッシュボードタブを追加する
  - タブバーの先頭（クイック入力の左）にダッシュボードボタンを配置する
  - タブコンテンツ領域にダッシュボード用のコンテナを追加する
  - タブ順序の定義にダッシュボードを含め、スライドアニメーションが正しく動作するようにする
  - 初期タブの検証ロジックにダッシュボードを追加する
  - _Requirements: 6.1, 6.3_

- [x] 1.2 ダッシュボードモジュールを作成し、エントリポイントを配線する
  - ダッシュボード専用モジュールを新規作成し、状態モジュールと定数モジュールのみをインポートする
  - メインの描画関数をエクスポートし、初期化モジュール経由でグローバルに公開する
  - タブ切替時にダッシュボードの描画関数が呼び出されるようにする
  - 見積データが存在しない場合は「データがありません」の案内メッセージを表示する
  - _Requirements: 8.1, 8.2, 8.3, 1.4, 6.2_

- [x] 2. KPIサマリーカードの実装
- [x] 2.1 主要KPIを集計してサマリーカードとして描画する
  - 見積データと実績データから総見積工数、総実績工数、差異（時間・割合）、全体進捗率を計算する
  - 4つのKPIをカード形式で並べて表示する。各カードにはラベル・数値・補足情報を含める
  - ダッシュボード固有のCSSクラス（プレフィックス付き）でカードのスタイルを定義する
  - タブ切替のたびに最新データで再計算・再描画する
  - _Requirements: 1.1, 1.2_

- [x] 2.2 KPIの変化方向インジケーターを追加する
  - 前回表示時のKPI値をローカルストレージに保存する
  - 次回表示時に前回値と比較し、増加・減少・変化なしの方向を判定する
  - 各サマリーカードに矢印アイコン（↑↓→）で変化方向を視覚表示する
  - ローカルストレージへのアクセスが失敗した場合はインジケーターを非表示にして描画を継続する
  - _Requirements: 1.3_

- [x] 3. 版数別進捗オーバービューの実装
- [x] 3.1 全版数の進捗率を計算してプログレスバー付きで一覧表示する
  - 見積・実績・残見積データから版数ごとのEAC（完了時見積）と進捗率を計算する
  - ステータス判定ロジックを実装する（完了・順調・注意（120%閾値）・超過）
  - 各版数をプログレスバーとステータスカラーで表示し、進捗率の昇順（遅れが上）にソートする
  - 版数データが存在しない場合は空状態メッセージを表示する
  - _Requirements: 2.1, 2.2, 2.4_

- [x] 3.2 版数クリックでタスク内訳をインライン展開する
  - 版数の行をクリックすると、その版数に含まれるタスク別の進捗一覧を展開表示する
  - 各タスクに見積・実績・残・ステータスを含める
  - 再クリックで折りたたむトグル動作を実装する
  - _Requirements: 2.3_

- [x] 4. アラート・注意喚起パネルの実装
- [x] 4.1 見積超過・異常値タスクを検出してアラートリストを描画する
  - 見積と実績を版数×タスク単位で比較し、実績が見積の120%以上のタスクを超過アラートとして抽出する
  - 乖離率が50%以上のタスクを異常値警告として抽出する
  - アラートを超過率の降順（重要度順）にソートして表示する
  - 各アラート項目に版数名、タスク名、見積→実績、超過率を含める
  - アラートが存在しない場合は「問題なし」の正常ステータスメッセージを表示する
  - _Requirements: 3.1, 3.2, 3.4, 3.5_

- [x] 4.2 アラート項目のクリックでタスク詳細をインライン展開する
  - アラート項目をクリックすると、該当タスクの工程別内訳や担当者情報を展開表示する
  - 再クリックで折りたたむトグル動作を実装する
  - _Requirements: 3.3_

- [x] 5. 担当者別ワークロードサマリーの実装
- [x] 5.1 担当者ごとのワークロードを集計してバー形式で描画する
  - 見積・実績データから担当者一覧を動的に抽出する
  - 各担当者の見積工数と実績工数を横棒バーで比較表示する
  - 見積精度（実績÷見積の割合）を数値で表示する
  - 実績がキャパシティ基準（月間人月工数）を超過している担当者を視覚的に警告表示する
  - _Requirements: 4.1, 4.2, 4.3_

- [x] 5.2 担当者クリックで実績詳細をインライン展開する
  - 担当者の行をクリックすると、その担当者のタスク別実績一覧を展開表示する
  - 各明細に版数、タスク名、工数、日付を含める
  - 再クリックで折りたたむトグル動作を実装する
  - _Requirements: 4.4_

- [x] 6. 直近の活動履歴フィードを実装する
  - 実績データを日付の降順でソートし、直近10件をリスト表示する
  - 各アクティビティに日付、担当者名、タスク名、工数を含める
  - 直近7日間に実績入力がない場合は「最近の入力がありません」の注意メッセージを表示する
  - _Requirements: 5.1, 5.2, 5.3_

- [x] 7. レスポンシブレイアウトとテーマ対応CSSの整備
- [x] 7.1 デスクトップとモバイルのレスポンシブレイアウトを整える
  - デスクトップ幅（768px以上）ではサマリーカードとパネルをグリッドレイアウトで配置する
  - モバイル幅（768px未満）では全セクションを縦一列に積み重ねる
  - 全CSSクラスにダッシュボード固有のプレフィックスが付いていることを確認し、名前空間の分離を担保する
  - CSSセクションをコメントブロックで囲み、削除時の範囲を明確にする
  - _Requirements: 7.1, 7.2, 8.4_

- [x] 7.2 全デザインテーマでのスタイル整合性を確保する
  - 5つのデザインテーマ（default, obsidian, paper, mosaic, graphite）それぞれに対してダッシュボードカード・パネルのオーバーライドを定義する
  - テーマカラー設定がサマリーカードに正しく反映されることを確認する
  - _Requirements: 7.3_
