# リスク分析レポート

> 作成日: 2026-02-27
> 対象: 工数管理システム（GitHub Public リポジトリ + GitHub Pages）

---

## 1. GitHub公開に関するリスク

### 1-1. ソースコード公開

| リスク | 深刻度 | 詳細 |
|--------|--------|------|
| ビジネスロジックの露出 | 中 | 工数計算ロジック・標準工数・人月換算の考え方が誰でも読める。競合他社や取引先が社内の工数管理方法を把握できる |
| 定数値からの組織情報推測 | 中 | `constants.js`の標準工数・稼働日数などから、勤務体系や組織の前提条件が推測可能 |
| 脆弱性の発見容易性 | 低 | 攻撃者がコードを読んで弱点を探せる（ただしサーバーレスで攻撃面が小さい） |

### 1-2. Git履歴

| リスク | 深刻度 | 詳細 |
|--------|--------|------|
| メールアドレスの公開 | 中 | 全コミットに個人メールアドレスが記録されている。スパムやフィッシングの標的になりうる |
| ローカルパスの公開 | 低 | `.mcp.json`に`/Users/kmori/...`のパスが記録済み。個人特定の材料になる |
| 開発環境情報の公開 | 低 | `.serena/`設定ファイルがコミットされている |

### 1-3. Git履歴チェック結果（2026-02-27実施）

**問題なし：**
- APIキー、パスワード、トークンなどの機密情報 → なし
- 秘密鍵、証明書 → なし
- 業務上の実データ（顧客名、社名など） → なし
- バックアップファイル（実データ入りJSON） → なし
- `.env`ファイル → なし

### 1-4. GitHub Pagesの公開

| リスク | 深刻度 | 詳細 |
|--------|--------|------|
| 意図しないユーザーの利用 | 低 | 誰でもアクセス可能（データは各ユーザーのブラウザに閉じる） |
| 実験ブランチの公開 | 低 | preview/sandboxもデプロイされるため、未完成機能が外部から見える |
| SEO経由での発見 | 低 | 検索エンジンにインデックスされ、意図せず広く知られる可能性 |

### 1-5. コード改ざんリスク

- Publicリポジトリでも、他人が直接pushやマージはできない
- Pull Requestを送ることは可能だが、マージ判断はオーナーのみ
- **GitHubアカウント乗っ取り時のみリスクあり** → 二要素認証(2FA)で対策

## 2. データ関連リスク

### 2-1. localStorage依存

| リスク | 深刻度 | 詳細 |
|--------|--------|------|
| データ消失 | **高** | ブラウザのキャッシュクリア、再インストール、OS初期化で全消失。バックアップなしでは復旧不可 |
| 容量制限 | 中 | localStorageは通常5〜10MB。長期運用でデータ量が増えると上限に達する可能性 |
| ブラウザ間共有不可 | 中 | ブラウザ移行やPC買い替え時にデータ移行が手動（JSONエクスポート/インポート） |
| 同一オリジン依存 | 中 | GitHub PagesのURLが変わるとデータにアクセスできなくなる |

### 2-2. バックアップ運用

| リスク | 深刻度 | 詳細 |
|--------|--------|------|
| バックアップ忘れ | **高** | 自動バックアップ機能がないため、手動エクスポートを忘れるとデータ喪失リスクが高い |
| バックアップファイルの管理 | 中 | エクスポートしたJSONファイルの保管場所・世代管理がユーザー任せ |

## 3. セキュリティリスク

### 3-1. クライアントサイド

| リスク | 深刻度 | 現状の対策 |
|--------|--------|-----------|
| XSS | 低 | `escapeHtml`/`escapeForHandler`実装済み、CSP設定あり |
| 悪意あるJSONインポート | 低 | データ構造検証あり、ユーザー確認ダイアログあり |
| 共有PCでのデータ閲覧 | 中 | 認証機能なし。同じブラウザを使う人は全データにアクセス可能 |
| CSPの`unsafe-inline` | 低 | 早期テーマ適用のために必要。外部スクリプト注入はCSPで防止 |

### 3-2. サプライチェーン

| リスク | 深刻度 | 現状の対策 |
|--------|--------|-----------|
| ライブラリの脆弱性 | 低 | `xlsx.mjs`と`japanese-holidays.js`はローカルバンドル済み。ただし更新が止まると既知脆弱性が放置される |
| Google Fontsの依存 | 低 | 唯一の外部接続。障害時はフォント読み込み失敗するが機能には影響なし |

## 4. 運用・事業継続リスク

| リスク | 深刻度 | 詳細 |
|--------|--------|------|
| 単一開発者依存 | **高** | 開発・保守が個人に依存。不在時はメンテナンス停止 |
| GitHub依存 | 中 | GitHubの障害・ポリシー変更・アカウント凍結でサービス停止 |
| スケーラビリティの限界 | 中 | チーム拡大時、localStorageベースではデータ共有不可。アーキテクチャ変更が必要 |
| ブラウザ互換性 | 低 | ES Modules/localStorage仕様変更で動作しなくなる可能性 |

---

## 5. 法的・コンプライアンスリスク

| リスク | 深刻度 | 詳細 |
|--------|--------|------|
| 業務データの公開リスク | 中 | スクリーンショットやデモデータに実際の業務情報を含めないよう注意 |
| ライセンス問題 | 低 | SheetJS（Apache 2.0）のライセンス準拠を確認すること |
| 個人情報保護 | 中 | 従業員名・工数データは個人情報に該当しうる。エクスポートファイルの取り扱いに注意 |
