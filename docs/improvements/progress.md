### 2026-04-14 — ✅ G-20 未入力日ハイライト

**ステータス**: ✅ 完了
**設計書**: [07-operations.md#20-1-未入力日のハイライト](./07-operations.md)
**メモ**: [memos/G-20.md](./memos/G-20.md)

**スクショ**:
- [Before (Desktop)](./screenshots/G-20-before-desktop.png)
- [After (Desktop)](./screenshots/G-20-after-desktop.png)
- [After (Mobile)](./screenshots/G-20-after-mobile.png)
- [休暇日除外テスト](./screenshots/G-20-vacation-test.png)
- [全担当者モードテスト](./screenshots/G-20-all-members-test.png)

**実装範囲**:
- 個人別カレンダー（実績一覧）に未入力営業日のハイライト機能を追加
- カレンダー上部に「未入力: N日」のサマリーバナーを表示
- 行レベルで斜線パターン + 左ボーダー + 「未入力」バッジで強調
- `js/actual.js`: 検出ロジック追加（+30行）
- `style.css`: スタイル追加（+85行）

**判断ポイント**（設計書からの変更）:
- **適用モード**: `member`ビューでのみ有効化（`all`では誰の未入力か曖昧になるため）
- **今日の扱い**: 未入力扱いにしない（1日の途中で赤表示は不快感を生む）
- **ハイライト**: 絵文字の代わりにアンバー系斜線パターン + ボーダー（既存テーマと調和）

**確認してほしい点**:
- [ ] ハイライトの視覚的な強さは適切か（強すぎ/弱すぎ）
- [ ] 「未入力: N日」バッジの位置・文言は適切か
- [ ] 個人別モード時のみ自動有効化する仕様は妥当か

**残作業（次回以降の別項目）**:
- クイック入力タブの「未入力サマリーウィジェット」
- 連続未入力日数の警告
- 「この日を入力」ボタンからのクイック入力遷移
- 管理者向け入力率ダッシュボード（G-20-3として独立項目）

**所要時間**: 約2時間（設計メモ→実装→ウォークスルー作成）
