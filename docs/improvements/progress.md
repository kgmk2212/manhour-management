### 2026-04-14 — ✅ H-24 入力バリデーション + 重複検出

**ステータス**: ✅ 完了（H-24-1, H-24-3 を統合実装）
**設計書**: [08-security.md#24-データ整合性](./08-security.md)
**メモ**: [memos/H-24.md](./memos/H-24.md)
**動作確認ガイド**: [walkthrough/H-24.html](./walkthrough/H-24.html) ⭐

**実装範囲**:
- 新規モジュール `js/validation.js` (+150行) でバリデーション基盤を構築
- 3つの入力経路（クイック入力/実績編集/その他作業）に組込
- エラー/警告を分離し、標準 `alert/confirm` で表示
- 重複検出（同日・同バージョン・同タスク・同工程・同メンバー）

**判定ルール**:
| 項目 | エラー | 警告 |
|------|--------|------|
| 1件の時間 | ≤0 / >24h | >12h |
| 1日合計 | >24h | >10h |
| 日付 | - | 未来日 |
| 重複 | - | 同組合せで既存 |

**判断ポイント**（設計書からの変更）:
- **複数警告の集約**: 設計書では個別ダイアログだったが、1つに集約（ダイアログ連鎖は鬱陶しい）
- **エラー/警告の分離**: 新設。errorsは阻止、warningsは確認
- **重複時の動作**: 3択→2択（追加/キャンセル）に簡素化

**確認してほしい点**:
- [ ] 閾値（12h/10h/24h）は妥当か
- [ ] 重複時の2択で十分か（3択「追加/上書き/キャンセル」が必要か）
- [ ] 標準confirmの見た目で許容か、カスタムダイアログ化すべきか

**残作業（次回以降の別項目）**:
- 見積入力のバリデーション（同様の基盤を流用可能）
- 全データ一括重複スキャン機能（H-24-1 後半）
- 見積 vs 実績の紐づけ検証（H-24-2）
- 打ち合わせ一括追加への組込（UX設計が必要）

**所要時間**: 約2.5時間

### 2026-04-14 — ✅ G-20 未入力日ハイライト

**ステータス**: ✅ 完了
**設計書**: [07-operations.md#20-1-未入力日のハイライト](./07-operations.md)
**メモ**: [memos/G-20.md](./memos/G-20.md)

- [After (Mobile)](./screenshots/G-20-after-mobile.png)
- [休暇日除外テスト](./screenshots/G-20-vacation-test.png)
- [全担当者モードテスト](./screenshots/G-20-all-members-test.png)

**実装範囲**:
- 個人別カレンダー（実績一覧）に未入力営業日のハイライト機能を追加
- カレンダー上部に「未入力: N日」のサマリーバナーを表示
- 行レベルで斜線パターン + 左ボーダー + 「未入力」バッジで強調
- `js/actual.js`: 検出ロジック追加（+30行）
- `style.css`: スタイル追加（+85行）

**判断ポイント**（設計書からの変更）:
- **適用モード**: `member`ビューでのみ有効化（`all`では誰の未入力か曖昧になるため）

- **ハイライト**: 絵文字の代わりにアンバー系斜線パターン + ボーダー（既存テーマと調和）

**確認してほしい点**:
- [ ] ハイライトの視覚的な強さは適切か（強すぎ/弱すぎ）
- [ ] 「未入力: N日」バッジの位置・文言は適切か
- [ ] 個人別モード時のみ自動有効化する仕様は妥当か

**残作業（次回以降の別項目）**:
- クイック入力タブの「未入力サマリーウィジェット」
- 連続未入力日数の警告
- 「この日を入力」ボタンからのクイック入力遷移
- 管理者向け入力率ダッシュボード（G-20-3として独立項目）
