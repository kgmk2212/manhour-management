G-20 · UX

未入力日ハイライト

実績入力を忘れている営業日を、カレンダー上で視覚的に明示する機能。 月末になって気づく事態を防ぎ、データの抜けに早期に対処できるようになります。

実装日 2026-04-14
所要時間 5分
影響範囲 実績一覧（カレンダー表示）
関連ファイル js/actual.js · style.css

このページの目的

実装した「未入力日ハイライト」機能が意図通りに動くかを、操作手順に沿って確認するためのガイドです。 各ステップは独立しているので、気になる動作だけを確認することもできます。

確認するシナリオ

  • 通常: 未入力営業日が強調表示される
  • 休暇: 個人休暇日は対象外になる
  • 全担当者モード: 強調表示が無効になる

事前準備

確認の前に、テストデータを用意します

0

テストデータの注入

ブラウザのコンソールで実行（DevTools → Console）

以下のコードをコンソールに貼り付けて実行すると、当月のテストデータが投入されます。 月の途中に意図的に「未入力日」を作るデータです。

// 当月のテストデータを注入 const today = new Date(); const year = today.getFullYear(); const month = today.getMonth(); const monthStr = String(month + 1).padStart(2, '0'); const estimates = [ { id: 'e1', version: 'v2.0', task: 'ログイン画面', process: 'PG', member: '森', hours: 40, workMonths: [`${year}-${monthStr}`] }, ]; const inputDays = [1, 2, 3, 6, 7, 8, 13, 14]; // 9, 10は意図的に未入力 const actuals = inputDays.map((d, i) => ({ id: `a${i}`, date: `${year}-${monthStr}-${String(d).padStart(2, '0')}`, version: 'v2.0', task: 'ログイン画面', process: 'PG', member: '森', hours: 8.0, createdAt: new Date().toISOString() })); localStorage.setItem('manhour_estimates', JSON.stringify(estimates)); localStorage.setItem('manhour_actuals', JSON.stringify(actuals)); localStorage.setItem('manhour_vacations', JSON.stringify([])); location.reload();
i 既存のデータをクリアします。残しておきたいデータがある場合は、先に「設定 → バックアップ」を実行してください。

シナリオ1: 基本動作

未入力営業日が強調表示されることを確認します

1

実績一覧 → 個人別カレンダーを開く

このシナリオで一番見たい画面に到達します

  • 1.1 サイドバーの 実績一覧 をクリック
  • 1.2 表示モードを 個人別 に変更
  • 1.3 担当者から 森 を選択
  • 1.4 表示形式が カレンダー になっていることを確認
2

未入力日のハイライトを確認

このスクリーンショットで強調表示されている要素を実画面と見比べる

1
2
3
  • 1
    サマリーバナー: 「未入力: 2日 4/9, 4/10」がオレンジの斜線パターン背景で表示。 一目で「何日抜けているか」「どの日か」が分かります。
  • 2
    未入力行のハイライト: 4/9と4/10の行に左側のオレンジバーと斜線パターンが付与。 土日の赤系とは色相が異なるため一目で区別できます。
  • 3
    「未入力」バッジ: 日付の隣に小さな色付きラベルで明示。 スクリーンリーダーでも「未入力」と読み上げ可能です。
確認チェック
  • 4/9（木）と 4/10（金）にオレンジ系の強調表示がある
  • 4/4（土）と 4/5（日）の赤系背景は変わっていない（既存仕様の維持）
  • 「未入力: 2日」のバナーが表示される
  • 未来日（今日より後）はハイライトされていない

シナリオ2: 休暇日の除外

個人休暇を登録した日が「未入力扱いされない」ことを確認

3

休暇を1件追加

4/9 を森の休暇日として登録

通常はクイック入力 → 休暇登録から行いますが、確認用に簡易コマンドも提供:

// 4/9 を森の休暇日として登録 const today = new Date(); const monthStr = String(today.getMonth() + 1).padStart(2, '0'); localStorage.setItem('manhour_vacations', JSON.stringify([ { id: 'v1', member: '森', date: `${today.getFullYear()}-${monthStr}-09`, type: '有給', hours: 8, description: 'テスト休暇' } ])); location.reload();

シナリオ3: 全担当者モードで非表示

「誰の未入力か」が曖昧になる全体表示では機能をオフにする仕様

5

表示モードを「全担当者」に切替

切替後、ハイライトが消えることを確認

  • 5.1 表示モードを 全担当者 に変更
  • 5.2 表示形式は カレンダー のまま
全担当者モード時の表示 サマリーバナー・行ハイライトともに非表示
確認チェック
  • 「未入力: N日」のバナーが表示されていない
  • 未入力日のオレンジ強調がない
  • 土日の赤背景は通常通り表示されている
i 仕様上の判断: 全担当者モードでは「誰が未入力か」が曖昧になるため、機能を意図的にオフにしています。 管理者向けの「入力率ダッシュボード」(G-20-3) を別途実装予定です。

追加で試したいエッジケース

    過去月: 「表示期間」を先月に切り替えて、過去のデータでも正しく動作するか
