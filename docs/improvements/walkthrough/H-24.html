実績入力時に「時間の異常値」「同日合計の超過」「未来日」「重複」を検出し、 保存前にユーザーに確認する仕組み。入力ミスによるデータ汚染を防ぎます。

このページの目的

入力バリデーション機能がケースごとに期待通り動作するかを、 具体的な入力例とそれに対するダイアログのモックアップで確認します。

検証するケース

  • エラー: 0時間、25時間、合計25時間超過
  • 警告: 15時間、未来日、同日合計11時間
  • 重複: 同じタスクの二重入力
  • 通常: 問題ない入力は素通り

判定の閾値

どのような入力が「エラー」「警告」となるか

項目 エラー（阻止） 警告（確認）
1件の時間 ≤ 0 / > 24h > 12h
1日の合計 > 24h > 10h
日付 — 未来日
重複 — 同日·同タスク·同工程·同メンバー

対応箇所

バリデーションが動作する3つの入力経路

  • A
    クイック入力タブ → 実績入力モード → 「追加」ボタン
    サイドバー「クイック入力」から。日常的な入力経路。
  • B
    実績編集モーダル → 保存ボタン
    実績一覧から行をクリック or 鉛筆アイコン。編集時も自身以外で重複判定。
  • C
    その他作業モーダル → 追加ボタン
    カレンダーから「その他」を追加する経路。
i 「打ち合わせ全員分一括追加」機能は今回は対象外です。一括追加のUXを壊さない設計にするため、個別の確認ダイアログが必要か検討が必要です。

テスト用データの準備

確認時に使う既存データ

重複や合計超過のテストには、ある程度の既存データが必要です。ブラウザのコンソールに以下を貼り付け:

// テスト用データ: 今日の実績を10h分登録 const today = new Date(); const ymd = `${today.getFullYear()}-${String(today.getMonth()+1).padStart(2,'0')}-${String(today.getDate()).padStart(2,'0')}`; localStorage.setItem('manhour_actuals', JSON.stringify([ { id: 1, date: ymd, version: 'v2.0', task: 'ログイン画面', process: 'PG', member: '森', hours: 3.0, createdAt: new Date().toISOString() }, { id: 2, date: ymd, version: 'v2.0', task: 'ログイン画面', process: 'PG', member: '森', hours: 7.0, createdAt: new Date().toISOString() } ])); localStorage.setItem('manhour_estimates', JSON.stringify([ { id: 'e1', version: 'v2.0', task: 'ログイン画面', process: 'PG', member: '森', hours: 40, workMonths: [`${today.getFullYear()}-${String(today.getMonth()+1).padStart(2,'0')}`] } ])); location.reload();

各シナリオの試し方:

  • 125h入力: クイック入力で時間を「25」→ 追加 → エラー確認
  • 2合計超過: 時間を「15」→ 追加 → エラー確認（既存10h + 15h = 25h）
  • 315h警告: 既存データを削除してから時間「15」→ 警告確認
  • 4未来日: 作業日を「2030-01-01」→ 警告確認
  • 5重複: 同じタスクを再度 時間「1」→ 重複警告確認

フィードバック

特に判断が必要なポイント

  • Q1: 閾値は妥当ですか？（12h警告、10h合計警告、24h上限）
    緩すぎる or 厳しすぎる場合は `js/validation.js` の `VALIDATION_THRESHOLDS` で調整可能
  • Q2: 重複時のダイアログは2択（OK/キャンセル）で十分ですか？
    設計書では「追加/上書き/キャンセル」の3択も検討されていました
  • Q3: 標準`alert`/`confirm`のUIを、カスタムダイアログで置き換えたほうが良いですか？
    今はOS標準。モバイルでも見やすいがデザイン統一感はない
