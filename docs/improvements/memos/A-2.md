## なぜやるか

毎日似たような実績（定例会議、継続中の開発タスク）を手入力するのは退屈で、
「週明けにまとめて入力したい」「昨日と同じ内容でOK」というケースが多い。
前営業日の実績を複製 → 必要箇所だけ編集する流れで、入力時間を劇的に削減する。

## 設計判断

### 採用する方針
- **起動導線**: クイック入力の実績フォーム下に控えめなdashed-borderボタン
  - 理由: 通常の入力フローを邪魔せず、必要な時だけ目に入る
- **コピー元のデフォルト**: 前営業日（土日スキップ）
  - 前営業日に実績がない場合は直近30日で実績がある日を探す
- **選択UI**: チェックボックスで個別選択。デフォルトは全て選択済み
  - よくあるのは「昨日と全く同じ」ケース
- **バリデーション**: H-24 の validation.js を流用
  - コピー時も未来日警告・重複警告が働く
  - エラーがあれば中止、警告は1回の確認ダイアログに集約

### 設計書から変更する点
| 項目 | 設計書 | 実装 | 理由 |
|------|--------|------|------|
| コピー元の日付指定 | 「前日」と「他の日からコピー」の2ボタン | 1ボタン+ドロップダウンで統一 | UIの複雑さを抑える |
| 時間編集 | コピー確認画面でインライン編集 | 同じ（設計書通り） | - |
| 選択UX | ☑チェックで選択 | 同じ | - |

## データフロー

```
[前日コピーボタン] クリック
    ↓
openCopyPreviousModal()
    ↓
モーダル表示:
  - コピー元日付: 前営業日（実績があれば） or 直近実績日
  - メンバー: クイック入力で選択中 or 最も直近のメンバー
  - 対象実績のチェックリスト表示（デフォルト全選択）
    ↓
ユーザー操作:
  - 日付・メンバー変更 → リスト再生成
  - チェック切替、時間編集
    ↓
[コピーして登録] クリック
    ↓
各実績に validateActualInput 実行
  - エラーあり → 中止（メッセージ表示）
  - 警告あり → 集約した1つの confirm で確認
  - OK → まとめて actuals.push
    ↓
Undoスタックに "addedAll" として記録
    ↓
UIを再描画・トースト通知
```

## 動作確認項目

### 基本動作
- [ ] 「前日の実績をまとめてコピー」ボタンが表示される
- [ ] クリックするとモーダルが開き、前営業日の実績が表示される
- [ ] デフォルトで全件チェック済み
- [ ] 「コピーして登録」で選択した件数が今日の日付で作成される

### 日付選択
- [ ] 前営業日（土日スキップ）がデフォルト
- [ ] コピー元ドロップダウンで過去30日の実績がある日を選択可能

- [ ] 「全員」を選択すると該当日の全メンバーの実績が表示される
- [ ] メンバーを切り替えるとリストが絞り込まれる

### 選択UI
- [ ] チェックボックスで個別選択/解除
- [ ] 「すべて選択」「すべて解除」ボタンが動作
- [ ] 解除した行は半透明で表示
- [ ] 時間を直接編集可能
- [ ] 選択件数と合計時間がリアルタイムで更新

### バリデーション連動
- [ ] コピー先で重複する場合に警告
- [ ] 未来日へのコピー時に警告
- [ ] 複数件の警告が1つのダイアログに集約される
- [ ] エラーがある件はコピーされない

### Undo
- [ ] 「Ctrl+Z」または変更履歴から、コピー作成した全件を元に戻せる

### モバイル
- [ ] モーダルが画面に収まる
- [ ] 縦積みレイアウトに切り替わる
