## なぜやるか

毎日似たような入力パターン（定例会議2h、レビュー1h）を繰り返し入力する手間が大きい。
パターンをテンプレート化 → ワンクリックで呼び出せれば、日次入力の時間が劇的に短縮する。

## データモデル

```javascript
// localStorageキー: manhour_templates
{
  templates: [
    {
      id: "tmpl_1234567890",
      name: "定例会議 (月曜)",         // ユーザー付与の名称
      version: "v2.0",                  // 空文字可（その他作業テンプレ）
      task: "定例会議",
      process: "",                      // 空文字可
      member: "森",
      hours: 2.0,
      usageCount: 15,                   // 利用回数
      lastUsed: "2026-04-14T10:00:00",  // 最終使用日時
      createdAt: "2026-03-01T00:00:00"
    }
  ]
}
```

## 設計判断

### 採用する方針
- **単一エントリ優先**: 「セット」機能は複雑度が跳ね上がるため次回
- **一覧 = モーダル**: フォーム直下に追加するとクイック入力画面が煩雑になる
- **自動ソート**: 利用頻度（usageCount降順、tie-break lastUsed降順）でソート
- **適用 = フォームに値をセット**: 即保存はしない（ユーザーが最終確認するため）

### 設計書から変更する点
| 項目 | 設計書 | 実装 | 理由 |
|------|--------|------|------|
| タブ位置 | 新規セグメント「テンプレ」 | モーダルで代用 | セグメント追加はレイアウト変更が大きい。将来的にセグメント化は可能 |
| アイコン | 視覚的識別子 | 省略 | MVP優先、後で追加可能 |
| frequency | daily/weekly/manual | 省略 | 利用頻度で十分。将来サジェスト機能と連動可 |
| テンプレートセット | 複数エントリ一括 | 未実装 | スコープ縮小 |

## データフロー

```
[テンプレート ボタン] クリック (クイック入力タブ)
    ↓
openTemplatesModal()
    ↓
モーダル表示:
  - 保存済みテンプレート一覧（利用頻度順）
  - 各テンプレに「適用」「編集」「削除」ボタン
  - 下部に「新規テンプレート作成」セクション
    ↓
ユーザー操作:
  [適用] → フォームに値をセット + モーダル閉じる + usageCount++
  [現在の入力をテンプレ保存] → 名前入力 → 保存 → 一覧更新
  [編集] → インライン編集
  [削除] → 確認 → 削除
```
