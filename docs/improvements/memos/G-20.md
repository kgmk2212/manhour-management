## なぜやるか

営業日にも関わらず実績入力を忘れている日を視覚的に明示し、月末になって気づく事態を防ぐ。
カレンダー画面を開くだけで「抜けている日」が即座に分かる状態を作る。

---

## 実装スコープ

### 今回やること
- [x] カレンダーグリッド（実績タブ）の未入力営業日を視覚的にハイライト
- [x] カレンダー上部に「未入力: N日」のサマリーバッジ表示
- [x] `detectMissingDays()` 関数の実装
- [x] 個人別表示（viewMode='member'）でのみ有効化

### 今回やらないこと（次回以降）
- クイック入力タブの「未入力サマリーウィジェット」（別コミット）
- 連続未入力日数の警告
- 「この日を入力」ボタンからのクイック入力遷移
- 管理者向け入力率ダッシュボード（G-20-3として別項目）

## 設計判断

### 採用する方針
設計書の基本ロジックに従う:
- 営業日（月〜金） - 休暇 - 祝日 - 会社休日 = 入力期待日
- 期待日のうち、実績0件 かつ 今日以前 = 未入力日

### 設計書から変更する点
| 項目 | 設計書 | 実装 | 理由 |
|------|--------|------|------|
| 適用モード | 明記なし | `member`ビュー時のみ | `all`ビューでは誰の未入力か曖昧になる |
| 今日自身の扱い | 明記なし | 未入力扱いにしない | 1日の途中で赤表示は不快感を生む |
| ハイライト色 | 🔴 絵文字 | 斜線パターン + 赤系ボーダー | 既存のQuiet Depthテーマに調和 |

## 動作確認項目

### 機能テスト
- [ ] member ビューで過去の未入力営業日にハイライトが表示される
- [ ] 祝日・会社休日はハイライトされない
- [ ] 個人休暇日はハイライトされない
- [ ] 土日はハイライトされない
- [ ] 今日および未来日はハイライトされない

- [ ] ヘッダーに「未入力: N日」が正しく表示される

- [ ] デスクトップで崩れない
- [ ] モバイル（375px）で崩れない
- [ ] 既存の祝日/土日表示と視覚的に競合しない

### 既存機能への影響

- [ ] 実績入力済みの日の表示は変わらない
