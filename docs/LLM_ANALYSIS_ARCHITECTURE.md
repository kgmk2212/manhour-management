## 2. データフロー詳細

### 全体の流れ

```
[Step 1]              [Step 2]           [Step 3]            [Step 4]
ユーザー操作           自動処理            LLMサーバー          自動処理
バックアップ保存   →   共有フォルダ    →   推論実行        →   結果を配信
                      へコピー            結果JSON生成         アプリサーバーへ

[Step 5]
ユーザー閲覧
レポート画面で
AI分析を表示
```

### Step 1: バックアップ保存（ユーザー操作）

```
ユーザーのブラウザ
    │
    │  既存のバックアップ機能
    │  autoBackup() → JSON出力
    │
    ▼
ローカルPC のダウンロードフォルダ
    │
    │  ファイル名: 工数管理_バックアップ_2026-02-24_09-00-00.json
    │
    │  内容:
    │  {
    │    estimates: [...],      // 見積データ
    │    actuals: [...],        // 実績データ
    │    remainingEstimates: [...],  // 残見積
    │    schedules: [...],      // スケジュール
    │    settings: {...},       // 設定
    │    timestamp: "...",
    │    version: "1.1"
    │  }
    │
    ▼
共有フォルダ (手動 or 自動で配置)
    backups/team-a/工数管理_バックアップ_2026-02-24_09-00-00.json
```

### Step 2: データ取り込みと要約生成（LLMサーバー）

```
共有フォルダ
    │
    │ rsync / cron (例: 毎晩 2:00)
    ▼
LLMサーバー: /data/backups/
    │
    │ 要約生成スクリプト (Python)
    │ summarize.py
    │
    │  処理内容:
    │  1. 最新のバックアップJSONを読み込む
    │  2. estimates, actuals から集計:
    │     - 全体の見積精度
    │     - バージョン別の進捗・精度
    │     - 工程別の精度
    │     - メンバー別のパフォーマンス
    │     - 月次トレンド
    │     - キャパシティ情報
    │     - 異常値タスク
    │  3. 要約JSONを出力
    │
    ▼
LLMサーバー: /data/summaries/
    team-a_summary.json
```

### Step 3: LLM推論（LLMサーバー）

```
要約JSON
    │
    │ 推論スクリプト (Python)
    │ analyze.py
    │
    │  処理内容:
    │  1. 要約JSONを読み込む
    │  2. システムプロンプト + 要約データでプロンプト構築
    │  3. Ollama API に送信
    │     POST http://localhost:11434/api/generate
    │     {
    │       "model": "llama3:8b",  ← モデルは検証後に決定
    │       "prompt": "...",
    │       "format": "json",
    │       "stream": false
    │     }
    │  4. レスポンスのJSONをパース・検証
    │  5. メタデータを付与して結果JSON出力
    │
    ▼
LLMサーバー: /data/results/
    team-a_analysis.json
    │
    │  内容:
    │  {
    │    "meta": {
    │      "generated_at": "2026-02-24T02:15:00+09:00",
    │      "model": "llama3:8b",
    │      "source_backup": "工数管理_バックアップ_2026-02-24.json",
    │      "analysis_period": "2025-10 ~ 2026-02"
    │    },
    │    "team_evaluation": { ... },
    │    "outlook": { ... },
    │    "recommended_actions": [ ... ],
    │    "next_review_focus": "..."
    │  }
```

### Step 4: 結果配信（自動処理）

```
LLMサーバー: /data/results/
    │
    │ rsync / cron (推論完了後に実行)
    ▼
アプリサーバー: /var/www/manhour/analysis/
    ├── team-a.json
    ├── team-b.json
    └── team-c.json
```

### Step 5: フロントエンド表示

```
ブラウザ (レポート画面)
    │
    │ fetch('/analysis/team-a.json')
    │
    │ → 200 OK: AI分析セクションを表示
    │   - チーム評価サマリー
    │   - 展望・予測
    │   - 推奨アクション一覧
    │   - 生成日時の表示
    │
    │ → 404 Not Found: AI分析セクションを非表示
    │   (graceful degradation)
```

## 3. LLMサーバーの構成詳細

```
LLMサーバー
├── /opt/ollama/              # Ollama本体
│   └── models/               # ダウンロード済みモデル
│       └── llama3-8b/        # (例)
│
├── /opt/manhour-analysis/    # 分析パイプライン
│   ├── summarize.py          # バックアップ → 要約JSON
│   ├── analyze.py            # 要約JSON → LLM推論 → 結果JSON
│   ├── run_pipeline.sh       # パイプライン実行スクリプト
│   ├── config.yaml           # 設定ファイル
│   │   ├── ollama_url        #   Ollama APIのURL
│   │   ├── model_name        #   使用モデル名
│   │   ├── teams             #   チーム一覧
│   │   └── prompts_dir       #   プロンプトテンプレート格納先
│   └── prompts/
│       └── team_analysis.txt # システムプロンプト
│
├── /data/
│   ├── backups/              # バックアップJSON (共有フォルダからコピー)
│   │   ├── team-a/
│   │   ├── team-b/
│   │   └── team-c/
│   ├── summaries/            # 要約JSON (中間生成物)
│   │   ├── team-a_summary.json
│   │   └── ...
│   └── results/              # LLM推論結果 (アプリサーバーへコピー)
│       ├── team-a_analysis.json
│       └── ...
│
└── /etc/cron.d/
    └── manhour-analysis      # cron設定
        # 0 2 * * * /opt/manhour-analysis/run_pipeline.sh
```

### パイプライン実行フロー

```
run_pipeline.sh
    │
    ├── 1. rsync: 共有フォルダ → /data/backups/
    │
    ├── 2. for each team:
    │      python summarize.py --team team-a
    │      python analyze.py --team team-a
    │      (チームごとに順次実行 → GPU負荷を分散)
    │
    ├── 3. rsync: /data/results/ → アプリサーバー
    │
    └── 4. ログ出力・完了通知
```

## 4. 定期実行のタイミング案

```
時刻    処理
─────────────────────────────────────────
02:00   バックアップをLLMサーバーにコピー
02:05   チームAの要約生成 + LLM推論
02:15   チームBの要約生成 + LLM推論 (※推論時間はモデル・データ量による)
02:25   チームCの要約生成 + LLM推論
 ...
03:00   全結果をアプリサーバーにコピー
─────────────────────────────────────────
09:00   チームリーダーが出社、レポート画面で最新のAI分析を確認
```

※ チーム数やモデルサイズに応じて所要時間は変動。
  7B モデルで要約データ入力の場合、1チームあたり数分〜10分程度を想定。

```

│                                                    │

│              ◀──rsync──── LLMサーバー               │
│                                                    │
│   LLMサーバー ──rsync──▶ アプリサーバー               │
│              (推論結果の配信)                        │
│                                                    │
│   LLMサーバー内部:                                  │
│   analyze.py ──HTTP──▶ localhost:11434 (Ollama)    │
│                                                    │
│   ★ 外部ネットワークへの通信: なし                    │
│                                                    │
└────────────────────────────────────────────────────┘
```
