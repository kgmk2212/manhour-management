# 工数管理システム - 開発ガイド

## プロジェクト概要
単一HTMLファイルで構成された工数管理システム。見積・実績の管理、レポート分析、担当者別の工数分析などの機能を提供。

## 技術スタック
- 純粋なHTML/CSS/JavaScript（フレームワークなし）
- ExcelJS（Excelファイル出力）
- Chart.js風の独自Canvas描画
- ローカルストレージによるデータ永続化

## 基本方針

### レスポンシブ対応
- モバイルファースト設計
- ブレークポイント: 768px（PC/モバイルの境界）
- PC表示とモバイル表示で適切にUIを調整
- グラフや表示要素はデバイスサイズに応じて最適化

### コード管理
- 単一ファイル（index.html）で完結
- インラインスタイルよりCSSクラスを優先（保守性向上）
- 関数は機能ごとに適切に分割

### Git運用
- コミットメッセージは日本語で記述
- フォーマット: `種別: 簡潔な説明`（例: `改善: PC表示時のドーナツグラフを大きく表示`）
- 必ず `Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>` を追加

#### コミット・プッシュの提案
- **修正完了後は必ずコミット・プッシュを提案する**
- git diff と変更内容から適切なコミットメッセージを自動生成
- 簡単な修正（UI調整、バグ修正、スタイル変更など）は確認なしで提案
- 複雑な修正（機能追加、ロジック変更、破壊的変更など）は内容を説明してから提案
- ユーザーは「はい」「お願いします」など簡単な返答で承認可能

## UI/UXガイドライン

### グラフ表示
- ドーナツグラフ: PC表示時はより大きく表示（280px）、モバイルは200px
- カラースキーム: テーマに合わせた配色、薄いパステルカラーを使用
- 凡例: 明確で読みやすく、パーセンテージを表示

### レポート機能
- 担当者別分析: 見積vs実績の比較、工数内訳の可視化
- インサイト機能: AIライクな分析コメントを表示
- 月別/全期間でのフィルタリング対応
