# smart-filter-hint — フィルタバーの「気づきやすさ」3案

## 背景・目的

実績一覧などで下にスクロールすると、上部のフィルタバー（`filter-bar-sticky`）が画面外へ出る。
現在の **hoverモード**（設定 > フィルタバー表示 = `hover`）では、再表示する手段が
「画面上端 40px にマウスを乗せる」だけで、知らないと気づけない。

この導線を**視覚的な合図**で気づきやすくする 3 案を、現在の実装に忠実なインタラクティブ
モックアップで比較する。

## 対象の実装

- `index.html` インラインの `smart-sticky-bar` 生成ロジック
  - `showFixedBar()` … smartSticky モード（スクロールで表示）
  - `showHoverBar()` … hover モード（上端ホバーで表示）
- 設定キー: `manhour_settings.filterBarMode` = `smartSticky` | `hover` | `off`

> 関連修正（同一機能・別タスク）:
> - `showHoverBar` に「選択中ボタンを中央へスクロール」処理を追加（選択月が見えない問題）
> - hover バーのチカチカ（表示/非表示の振動）解消

## 3 案

| 案 | 概要 | 長所 | 短所 |
|----|------|------|------|
| **A. 上端ハンドル** | 画面上端中央に小さなつまみを常時表示。ホバー/クリックで展開 | アフォーダンスが明確・タッチ可 | 最上部でもわずかに見える |
| **B. 上端インジケータ帯＋ホバー** | フィルタが隠れている間だけ上端に細いアクセント帯をやわらかく明滅。付近のホバーで展開 | 画面を塞がない・最小限の主張・既存ホバー操作と地続き | 帯が細くタッチでは狙いにくい |
| **C. 右上 FAB** | 画面右上に小さなフィルタアイコン。クリックで開閉 | 明確に目に入る・タッチで押しやすい | 前面に出てコンテンツに重なる |

**暫定選好: 案B（上端インジケータ帯＋ホバー）。** タッチ環境では案Cの併用が安全。

## 動かし方

`index.html` をブラウザで直接開く（外部依存なし・self-contained）。
各デバイス枠の中を下にスクロール → ヒント出現 → ホバー（案Cはクリック）で展開。
