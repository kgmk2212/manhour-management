# Excel ファイルを読み込んで追加 — 競合解決UI モックアップ

## 背景

これまで Excel は「出力のみ」「JSON バックアップ復元」は全データ上書きのみ、と「追加」のフローが存在しなかった。
Excel ファイルから見積・実績を **既存データを保持したまま追加** できる機能が要望された。

既存データと**競合**する行（見積で `(版数, 対応名, 工程, 担当)` が一致するなど）をどう扱うかが
ユーザー体験を大きく左右するため、競合解決 UI を 4 案検討して比較する。

## 検討日

2026-05-23

## 関連する確定済み方針

- **スコープ**: 見積・実績の両方をサポート。シート単位で取り込み対象を選択可能
- **見積の競合判定キー**: `(版数, 対応名, 工程, 担当)` 一致
- **実績の競合判定キー**: `(日付, 担当, 版数, 対応名, 工程, 工数)` の完全一致のみ（同ファイル二重取り込み事故防止）
- **モード**: 追加（append）のみ。全入れ替えしたい時は JSON バックアップ復元を使う
- **競合解決の方針**: プレビュー画面でユーザーが選択（4 案はその UI バリエーション）

## 共通シナリオ（4 案で同一）

- 取り込みファイル: `工数管理_エクスポート_2026-05-20_14-03-12.xlsx`
- 見積シート: 12 行（新規追加 8 / 競合 4）
- 実績シート: 30 行（新規追加 28 / 完全重複スキップ 2）

## 採用案

**案 D を採用**。理由：

- Excel 読み込みは「日次の作業」ではなく「不定期にまとまった件数を取り込む」シーン中心。
  操作頻度が低いほど「これから何が起きるか」が一画面で見えることの価値が高い
- サマリで「合計 39 件を反映」と明示することで、見積・実績の合計件数を取り違える事故を防げる
- 一括トグルで素早い判断、必要時のみ個別切替、最後にフッター数値で最終確定 — 3 段階のセーフティネット
- 実装コストは高いが、A〜C のサブ要素を組み合わせているだけで設計判断としては最終形

不採用案も削除せず Git に残すことで、将来「もっとシンプルに戻したい」となった時の検討材料として参照可能。
