Excel ファイルを読み込んで追加 — 競合解決UI 4 案

Excel から見積・実績を「追加」で取り込む際に、既存データと競合する見積行をどう扱うかの UI 案。 共通シナリオ：見積 12 行（追加 8 / 競合 4）+ 実績 30 行（追加 28 / 完全重複 2）。 最終的に 案 D を採用予定。他 3 案は設計判断の記録として保持する。

案 A
コンパクト一覧 + 行ラジオ
競合行を 1 行ずつ詰めて表示。上部の一括トグルで全選択／個別判断を切替。 判断を素早く済ませたい運用向け。
📐 一覧重視 ⚡ 実装中
案 B
シンプル二択ダイアログ
個別選択を捨て「全部上書き / 全部スキップ」のみ。最も学習コストが低く実装も最軽量。 細かい運用には不向き。
🪶 最小 ⚡ 実装軽
案 C
既存 vs 読み込み 並列比較
各競合行で左右に並べて差分強調。行ごとに上書き / スキップ選択。 視認性は最高だが、件数が多いとスクロール量が増える。
🔍 差分明確 ⚡ 実装中
案 D
サマリ + 並列比較 + 一括トグル
上部に「追加 / 競合 / 上書き / スキップ」の全体サマリ、続いて一括トグル、 各競合行は並列比較しつつ行単位で選択可。誤操作リスク最小。
✓ 全部入り ⚡ 実装重
